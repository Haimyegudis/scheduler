# תכנון: מערכת שיבוץ משמרות לטכנאים

**תאריך:** 2026-07-21
**סטטוס:** מאושר על ידי המשתמש

## מטרה

אפליקציית Web לניהול אילוצים ושיבוץ משמרות שבועי לטכנאים. שני סוגי משתמשים: טכנאי ומנהל. המנהל מייצר תוכנית שיבוץ אוטומטית לפי אילוצים, עורך ידנית, ומפרסם.

## החלטות מפתח (מאישור המשתמש)

- **פריסה:** אינטרנט, חינמי. Next.js (App Router) על Vercel + Neon Postgres (free tier).
- **גודל צוות:** 13–20 טכנאים.
- **ימי עבודה:** ראשון–חמישי. שישי אופציונלי — המנהל מדליק לשבוע ספציפי. שבת לא מאוישת.
- **מחסור בטכנאים:** עמדות נשארות ריקות ומסומנות באדום; המנהל פותר ידנית. אין שיבוץ בניגוד לאילוצים אוטומטית.
- **טכנאים רואים את התוכנית שפורסמה** — הטבלה המלאה, משמרות שלהם מודגשות.
- **עמדות זהות** — "עמדה 1–4", כל טכנאי מתאים לכל עמדה.
- **שפה:** עברית, RTL, רספונסיבי למובייל.

## ארכיטקטורה

- **Next.js** אפליקציה אחת: React frontend + API routes (server actions / route handlers).
- **DB:** Postgres (Neon בענן; SQLite/Postgres מקומי לפיתוח). ORM: Prisma.
- **Auth:** JWT ב-httpOnly cookie. טכנאים — הרשמה/התחברות במייל+סיסמה (bcrypt hash, מינימום 8 תווים, ללא אימות מייל). מנהל — קרדנציאלים קבועים `admin` / `admin123` (נבדקים בשרת), role=admin בטוקן.
- **זיהוי שבוע:** לפי תאריך יום ראשון של השבוע (`weekStart`, ISO date). ניווט חצים = ±7 ימים.

## מודל נתונים

### Technician
- `id`, `name`, `email` (unique), `passwordHash`, `createdAt`

### Constraint
- `id`, `technicianId`, `date` (ISO date), `value`: `morning | evening | flex | off`
- unique על (`technicianId`, `date`)

### Schedule (תוכנית שבועית)
- `id`, `weekStart` (unique), `status`: `draft | published`, `includeFriday` (boolean), `createdAt`, `updatedAt`

### Assignment
- `id`, `scheduleId`, `date`, `shift`: `morning | evening`, `station`: 1–4, `technicianId`
- unique על (`scheduleId`, `date`, `shift`, `station`)

## מסכים

### טכנאי
1. **הרשמה** — שם, מייל, סיסמה (8+). שגיאה ברורה על מייל תפוס.
2. **התחברות** — מייל + סיסמה.
3. **אילוצים שבועיים** — חצים בין שבועות; ימים ראשון–חמישי (+שישי אם הודלק בתוכנית של אותו שבוע); לכל יום 4 כפתורי בחירה: בוקר/ערב/גמיש/חופש; ברירת מחדל: לא נבחר (נחשב "לא מילא"); שמירה אוטומטית בכל שינוי. אם התוכנית לשבוע פורסמה — תצוגה נעולה + הסבר.
4. **צפייה בתוכנית** — אם קיימת תוכנית `published` לשבוע: טבלה מלאה (ימים × משמרות × עמדות), משמרות של הטכנאי מודגשות. אם אין — הודעה "טרם פורסמה תוכנית".

### מנהל
1. **התחברות** — username + password.
2. **לוח בקרה (לשבוע נבחר, חצים)** —
   - רשימת טכנאים: שם + סטטוס מילוי (מלא = כל ימי העבודה של השבוע; חלקי; לא מילא).
   - טבלת אילוצים: שורות=טכנאים, עמודות=ימים, תא=ערך האילוץ (צבע לכל סוג).
3. **תוכנית משמרות** —
   - מתג "כולל שישי" לשבוע. שינוי המתג נשמר מיידית (נוצרת רשומת Schedule בסטטוס טיוטה, גם ללא שיבוצים) — כך טכנאים רואים את שישי במסך האילוצים עוד לפני יצירת התוכנית.
   - כפתור **"צור תוכנית"** — מריץ אלגוריתם; אם קיימת טיוטה, מבקש אישור לדריסה.
   - טבלה: לכל יום, בוקר (עמדות 1–4) וערב (עמדות 1–4). כל תא = dropdown לבחירת טכנאי. ה-dropdown מציג זמינים לפי אילוץ; בחירה של לא-זמין או כפול-ביום אפשרית עם אזהרה מוצגת.
   - עמדה ריקה = רקע אדום.
   - כפתורים: **שמור טיוטה**, **פרסם**. אחרי פרסום עדיין ניתן לערוך ולפרסם מחדש.
   - שורת סיכום: מספר משמרות לכל טכנאי בשבוע (לבדיקת איזון).

## אלגוריתם השיבוץ

קלט: `weekStart`, `includeFriday`, אילוצי כל הטכנאים לשבוע.

לכל יום (ראשון–חמישי, +שישי אם פעיל), לכל משמרת בסדר בוקר→ערב:

1. **זמינים:** טכנאי זמין למשמרת אם האילוץ שלו לאותו יום הוא ערך המשמרת או `flex`. `off` או חסר → לא זמין. מי שכבר שובץ באותו יום — נפסל.
2. **מילוי 4 עמדות:** בוחרים בכל פעם את הזמין עם מספר המשמרות המצטבר הנמוך ביותר בשבוע (שוויון → מי ששובץ פחות במשמרת מהסוג הזה, אחר כך אקראי דטרמיניסטי לפי id). בבוקר, בשוויון בין `morning` ל-`flex` — עדיפות ל-`morning` כדי לשמור גמישים לערב.
3. **מחסור:** פחות מ-4 זמינים → עמדות נותרות ריקות (`technicianId = null` לא נשמר; פשוט אין רשומת Assignment לאותה עמדה).

חוקים קשיחים שהאלגוריתם לעולם לא מפר: טכנאי אחד לכל עמדה-משמרת; אין טכנאי פעמיים באותו יום; שיבוץ רק לפי אילוצים. עריכה ידנית של המנהל רשאית לחרוג — עם אזהרה ב-UI בלבד.

## API (route handlers)

- `POST /api/auth/register` — name, email, password → יצירת טכנאי + cookie.
- `POST /api/auth/login` — email, password → cookie.
- `POST /api/auth/admin-login` — username, password → cookie (role=admin).
- `POST /api/auth/logout`
- `GET /api/constraints?weekStart=` — אילוצי הטכנאי המחובר.
- `PUT /api/constraints` — עדכון אילוץ בודד { date, value }. נדחה אם התוכנית לשבוע פורסמה.
- `GET /api/schedule?weekStart=` — לטכנאי: רק אם `published`. למנהל: תמיד.
- **מנהל בלבד:**
  - `GET /api/admin/overview?weekStart=` — טכנאים + סטטוסים + טבלת אילוצים.
  - `POST /api/admin/schedule/generate` — { weekStart, includeFriday } → יצירת/דריסת טיוטה.
  - `PUT /api/admin/schedule` — עדכון שיבוצים ידני + includeFriday (שמירת טיוטה).
  - `POST /api/admin/schedule/publish` — { weekStart } → status=published.

## טיפול בשגיאות

- מייל תפוס → 409 + הודעה בעברית.
- סיסמה קצרה → ולידציה בצד לקוח ובשרת.
- עריכת אילוץ לשבוע שפורסם → 409 + נעילת UI.
- גישה ללא הרשאה (טכנאי ל-API של מנהל) → 403.
- אין מספיק זמינים → לא שגיאה; תוצאה חלקית + סימון אדום.
